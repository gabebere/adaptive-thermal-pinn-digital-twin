# Paper-data DeepXDE adaptive heat PINN

This folder contains the DeepXDE implementation and its dependencies.

The selected Axioms 12 (2023), 416 reference data describes a **2D** heat
problem, so the paper-compatible network has inputs `(X, Y, tau)`. Reducing it
to one spatial dimension would no longer reproduce the cited paper problem.

## Reference data

The script reads the reference CSV files without modifying them from the
repository-relative directory:

```text
../../Literature_results/mdpi_416_tables
```

- `table_N_40_times.csv`: 40 center-point values used for sequential adaptation.
- `table_N_study_times.csv`: eight printed literature values used for RMSE.
- The comparison location is `(X,Y)=(0.5,0.5)`.
- Eight new time instances are assimilated during each of five updates.

## PyTorch-comparison configuration

This copied version uses three 48-unit tanh hidden layers, 1,200 baseline Adam
iterations, and five adaptive windows of 160 iterations each. Learning rates,
physics/data sampling counts, and data-loss weight match the full PyTorch PINN
where the two frameworks expose equivalent settings. The network still has
three inputs because this workflow solves the paper's 2-D problem.

The comparison also uses Glorot uniform initialization, pseudorandom training
interior/initial points resampled every Adam iteration, and PyTorch-style loss weights: 1 for
the PDE, 2 for each of the four boundaries, 5 for the initial condition, and
10 for accumulated paper observations. Published observations remain at the
paper midpoint because no equivalent off-center literature measurements exist.

No reference or experimental data is generated by this program.

## Install and run

From this folder:

```powershell
python -m pip install -r requirements.txt
python online_heat_pinn.py --case table_1
```

Available cases are `table_1`, `table_2`, and `table_3`. A different reference
directory can be supplied with `--reference-dir`.

## Outputs

After training, the following files are written under the `results/` folder
beside `online_heat_pinn.py`, regardless of the terminal working directory:

- `literature_vs_pinn.csv`
- `baseline_vs_adaptive_center.csv`
- `literature_pinn_metrics.json`
- `paper_reference_comparison.png`
- `streaming_sensor_windows.png`
- `baseline_adaptive_error_comparison.png`

The JSON file records

```text
RMSE = sqrt(mean((theta_PINN - theta_literature)^2))
```

over the eight printed literature values, along with the adaptive batch errors.
